# Лог архитектурных решений

## 2026-02-28: Гибридный подход к стилям

**Решение:** CSS Custom Properties для дизайн-токенов + SCSS maps для вариантов компонентов.

**Причина:** CSS Custom Properties поддерживают темизацию через `:root` / `[data-theme]` без перекомпиляции. SCSS maps дают типобезопасность и генерацию вариантов через `@each`. Чистый CSS не умеет итерировать по map — поэтому гибрид.

## 2026-02-28: Брейкпоинты в SCSS, не в :root

**Решение:** Брейкпоинты остаются SCSS-переменными в `var.scss`.

**Причина:** CSS Custom Properties не работают внутри `@media (min-width: ...)`. Это ограничение спецификации CSS. Поэтому `$breakpoint-tablet: 768px` — единственный вариант.

## 2026-02-28: Компонентные конфиги рядом с .vue

**Решение:** `_config.scss` лежит в папке компонента (`UI/MyBtn/_btn.config.scss`), а не в `assets/styles/`.

**Причина:** Колокация — стили компонента рядом с компонентом. Проще находить, проще удалять. Глобальные стили (токены, миксины) остаются в `assets/styles/`.

## 2026-02-28: interface/ отдельно от types/

**Решение:** `app/types/` для TypeScript `type`, `app/interface/` для TypeScript `interface`.

**Причина:** Разделение по виду конструкции. Позволяет быстро находить нужное.

## 2026-02-28: Pinia Options API

**Решение:** Сторы используют Options API (`defineStore("name", { state, getters, actions })`).

**Причина:** Явная структура, легче читать для новых разработчиков. Setup API мощнее, но менее структурирован.

## 2026-02-28: Одна тёмная тема в _dark.scss

**Решение:** Не создавать отдельную папку `themes/` с `light.scss` и `dark.scss`.

**Причина:** Светлая тема — это дефолтные значения в `_tokens.scss`. Тёмная — переопределение семантических токенов в `_dark.scss`. Одного файла достаточно. Папка `themes/` оправдана только при 3+ темах.

## 2026-02-28: Миксины в отдельных файлах

**Решение:** Папка `mixins/` с файлами по категориям (`_font.scss`, `_interactive.scss`, `_responsive.scss`, `_typography.scss`) и `_index.scss` как точка входа.

**Причина:** Один файл `mixins.scss` разрастается. Разделение по ответственности + один `@use "mixins" as *` для подключения.

## 2026-02-28: Скил ui-ux-design отдельно от design-system

**Решение:** Два отдельных скила: `design-system` (архитектура токенов, dark mode, связь с конфигами) и `ui-ux-design` (визуальное качество — padding, radius, цветовая гармония, состояния, доступность).

**Причина:** design-system отвечает за «как хранить и связывать значения», ui-ux-design — за «какие значения выбрать чтобы было красиво и удобно». Разные ответственности, разные триггеры использования.

## 2026-02-28: Unitless line-height

**Решение:** Токены `--lh-*` хранят безразмерные значения (1.25, 1.5, 1.625) вместо px.

**Причина:** Безразмерный line-height масштабируется с font-size — стандартная рекомендация MDN и WCAG. При px-значениях текст с разным font-size может наползать или иметь избыточные отступы.

## 2026-03-16: Отказ от Swiper Thumbs в ProductGallery

**Решение:** Откат к простой галерее (десктоп: основное фото + кликабельные миниатюры через `activeImageIndex`, мобилка: Swiper + Pagination). Без модуля Thumbs.

**Причина:** Swiper Thumbs при связке двух экземпляров (основной + превью) вычисляет ширину слайдов некорректно (~28 млн px). Проблема воспроизводится при любом порядке инициализации (thumbs первым, v-if на готовность, ClientOnly). Вероятная причина — конфликт SSR/hydration или особенности расчёта размеров в Nuxt.

## 2026-03-17: ProductGallery без Swiper + zoom через CSS background

**Решение:** Полностью убрать Swiper из ProductGallery и со страницы товара. Галерея: единый layout (основное фото + миниатюры) для всех экранов. Zoom при наведении: приближённый фрагмент рядом с курсором, реализован через `background-image` и CSS-переменные (`--zoom-bg`, `--zoom-bg-size`, `--zoom-bg-pos`). Маппинг координат с учётом `object-fit: cover` (naturalWidth/naturalHeight, scale, offset).

**Причина:** Swiper на странице товара вызывал проблемы (гигантская ширина на мобильном, некорректная работа на десктопе). Реализация zoom через `<img>` давала пустой/тёмный блок — переход на CSS background решил проблему рендеринга.

## 2026-03-17: Удаление Vuetify

**Решение:** Полностью удалить Vuetify (vuetify, vite-plugin-vuetify, @nuxtjs/vuetify, плагин, конфиг).

**Причина:** Vuetify не использовался — ни один `v-*` компонент не применялся в шаблонах. Весь UI построен на кастомных компонентах (My*, App*). Vuetify подгружал глобальный CSS-reset, который потенциально конфликтовал со Swiper (ширина слайдов ~28 млн px). Темизация реализована кастомным `useTheme` composable без Vuetify.

## 2026-03-17: Swiper CSS глобально + ClientOnly

**Решение:** CSS Swiper (`swiper/css`, `swiper/css/pagination`) вынесен в `nuxt.config.ts` → `css[]`. Все `<Swiper>` обёрнуты в `<ClientOnly>`.

**Причина:** Локальный `import "swiper/css"` внутри компонентов мог загружаться с задержкой — Swiper инициализировался до применения своего CSS (`overflow: hidden` на `.swiper`). Без этого правила flex-контейнер `.swiper-wrapper` растягивал родителя до суммарной ширины всех слайдов. `<ClientOnly>` защищает от SSR-расчётов при отсутствии реального DOM.

## 2026-03-17: Swiper min-width: 0 для flex/grid

**Решение:** Глобально для `.swiper` задать `min-width: 0`, `min-height: 0`, `width: 100%`, `max-width: 100%` в `main.scss`. Дополнительно: `width: 100%`, `min-width: 0` для `gallery__mobile`, `gallery__swiper` (ProductGallery) и `similar__swiper` ([slug].vue).

**Причина:** В flex/grid контейнерах `min-width: auto` по умолчанию не даёт Swiper сжиматься, что приводит к гигантским размерам (~28 млн px) при расчёте ширины слайдов. Фикс из nolimits4web/swiper#3599.

## 2026-03-16: Детерминированный generateSizes

**Решение:** Заменить `Math.random()` на детерминированный хэш на основе размера (`(s * 2654435761) >>> 0) % 100`).

**Причина:** `Math.random()` даёт разные результаты на сервере и клиенте → hydration mismatch в ProductSizes (disabled/class расхождения).

## 2026-03-16: Автоскролл к форме заказа — двойной rAF + динамическая высота хедера

**Решение:** `nextTick → requestAnimationFrame → requestAnimationFrame`, затем `getBoundingClientRect` минус `header.offsetHeight` минус 20px.

**Причина:** Один `nextTick` или один `rAF` недостаточно — браузер не завершает layout. Высота хедера динамическая (top-bar скрывается при скролле), поэтому `scroll-margin-top` с фиксированным значением не подходит.

## 2026-02-28: Функции до 20 строк

**Решение:** Мягкий лимит 20 строк на функцию, допустимо до 30-40.

**Причина:** Читаемость, тестируемость, принцип единой ответственности. Чистые функции — в `utils/`, реактивные — в `composables/`.
